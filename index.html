Two systems that don't speak the same language.
This is the interpreter.

SAP Business One tracks the sales order. Wrike tracks the fulfillment task. Neither knows the other exists. Sync Hub simulates the middleware layer that keeps them honest — polling, transforming, retrying, and logging every hop in between.

How this actually gets built

This page is a simulation. The repo also ships the real pieces this kind of integration needs in production.

C#

Webhook receiver & retry policy

Handles inbound SAP B1 change events, applies exponential backoff on transient Wrike API failures, and dead-letters anything that fails after 3 attempts.

View IntegrationService.cs →
TypeScript

Field mapping schema

Typed contract between SAP B1's order fields and Wrike's custom fields, so a schema change fails a build instead of failing silently at 2am.

PowerShell

Ops health check

Scheduled task that checks queue depth, flags stuck records older than 15 minutes, and posts a summary to a Slack webhook.
